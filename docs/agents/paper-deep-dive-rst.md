# RTD 论文精解 RST 严格提示词

你要把指定 Journal Paper 制作为 WOEAI RTD 的“论文精解”页面。

这不是公众号导读、不是论文摘要、不是二次综述，也不是只提炼重点。目标是：

1. 以论文 PDF / 已批准作者稿为唯一正文来源。
2. 按原论文顺序提供完整中文译文。
3. 保留公式、图、表、章节、附录、文内引用和参考文献链。
4. 输出可直接放入 `docs/source/paper-notes/<publication_ref>.rst` 的 Sphinx RST。

不要编造论文没有的信息。不要用自己的理解替代原文。不要为了“可读性”删减句子、合并段落或跳过看似不重要的内容。

## 输入

开始前确认你拥有以下输入：

- `publication_ref`
- 论文 PDF 或已批准作者稿路径
- `docs/source/Publications.rst` 中对应的 `.. _<publication_ref>:` 条目
- 可公开使用的图片输出目录：`wechat/assets/public-safe/<publication_ref>/`

如果缺少 PDF / 作者稿，停止并报告缺失；不要只根据摘要或已有公众号文章生成 RTD 论文精解。

## 输出位置与页面开头

- 输出文件：`docs/source/paper-notes/<publication_ref>.rst`
- 页面顶部保留锚点：

  ```rst
  .. _paper-note-<publication_ref>:
  ```

- 页面标题使用中文精解题名，并包含“论文精解”。
- 标题下方预留精简版微信公众号文章链接；没有已发布链接时写：

  ```rst
  精简版微信公众号文章：待发布
  ```

- 可加入本页目录：

  ```rst
  .. contents:: 本页目录
     :local:
     :depth: 2
  ```

## 翻译强制规则

必须逐句翻译整篇论文正文。这里的“整篇论文正文”包括但不限于：

- 题名、作者、期刊信息、接收/修回/录用/在线发表信息
- 摘要
- 关键词
- 符号表、缩写表、nomenclature
- 引言
- 方法、理论、模型、算法、算例设置
- 结果、讨论、验证、敏感性分析
- 结论
- CRediT 作者贡献声明
- 利益冲突声明
- 数据可用性声明
- 致谢
- 附录
- 参考文献
- 全部图题、表题、图注、表注、脚注和必要的图表内文字

执行时遵守：

- 按 PDF 中出现的顺序处理，不得重排章节。
- 保留原章节编号，例如 `1 引言`、`3.2 新入流湍流生成器的推导`、`附录 A ...`。
- 原文一个自然段翻译为一个中文自然段。不要把多个原文段落合并成一个段落。
- 原文一个句子应对应一个中文句子或一个清晰可追溯的中文分句组。不要把一整段压缩成概述。
- 对列表、定义、定理、步骤、条件、枚举项，逐项翻译并保留原顺序。
- 不确定术语首次出现时可写“中文译名（English term, abbreviation）”，后文使用统一中文译名或约定缩写。
- 原文的限制、假设、负面结果、失败情形和适用边界必须翻译，不得只保留正面结论。
- 出版商页眉页脚、页码、下载信息、版权页脚、水印、广告和网页导航不属于论文正文，可不翻译。

## 严禁行为

不要出现以下行为：

- 不要写“本文主要介绍……”来代替原文段落翻译。
- 不要只翻译摘要、结论和图表附近文字。
- 不要跳过文献综述段落。
- 不要跳过参数设置、边界条件、网格设置、实验设置或数值设置。
- 不要跳过附录。
- 不要省略参考文献列表。
- 不要把多个公式之间的推导文字压缩成一句话。
- 不要删除文内引用，例如 `[12]`、`(Smith, 2020)`、`Fig. 3`、`Table 2`、`Eq. (5)`。
- 不要留下 `略`、`同上`、`此处省略`、`待补充`、`pending`、`TODO`、`to be confirmed`。
- 不要记录本机私有绝对路径、抓取日志、凭据、cookie、API payload 或私有预览链接。

## 图、表、公式与引用

### 公式

- 所有原文公式必须保留，并放在原文对应位置附近。
- 公式用 Sphinx `.. math::` 或行内 `:math:`，不要用公式截图替代可编辑公式。
- 保留原公式编号。
- 本项目 RTD 论文精解中禁止使用 `\tag{N}`，因为多行环境里 MathJax/Sphinx 可能报错。统一用右侧手动编号：

  ```rst
  .. math::

     E = mc^2 \qquad (1)
  ```

- 多行公式仍使用 `aligned`、`cases`、`bmatrix` 等 LaTeX 环境，但编号写成 `\qquad (N)`，不要写 `\tag{N}`。
- 公式前后的解释文字必须逐句翻译，不得只保留公式。

### 图片

- 从 PDF / 作者稿提取原图，优先保留原文最高可用清晰度。
- 图片放入 `wechat/assets/public-safe/<publication_ref>/`。
- RST 中使用相对路径，不得引用本机私有绝对路径。
- 每张图按原文出现顺序保留。确因版权、清晰度或源文件问题不能放图时，必须在交付说明中列出缺失图号和原因；不要在公开 RST 中写私有说明。
- 图题、图注、图中关键说明文字翻译为中文，保留原始图号，例如：

  ```rst
  .. figure:: ../../../wechat/assets/public-safe/<publication_ref>/fig03-example.png
     :alt: 图3 中文图题
     :align: center
     :width: 100%

     **图 3** 中文图题。中文图注逐句翻译原文说明。
  ```

### 表格

- 表题、表注、列名、行名和表内文字必须翻译。
- 简单表格优先写成 RST list-table / grid table。
- 复杂大表可作为高清图片插入，但表题、表注和关键表内文字仍要在 RST 中可读地翻译或说明。

### 文内引用与参考文献

- 文内引用必须保留，并能对应到末尾参考文献。
- 参考文献列表必须完整保留。英文参考文献可保留英文原文，不要翻译作者名和题名到无法检索的形式。
- 参考文献顺序、编号和 DOI/URL 信息尽量与原文一致。
- 不要删除引用链；如果正文引用了某条参考文献，末尾必须能找到对应条目。

## RST 结构要求

推荐结构如下，但必须以原论文结构为准，不得为套模板而删改原文：

```rst
.. _paper-note-<publication_ref>:

中文论文精解题名
================

精简版微信公众号文章：待发布

.. contents:: 本页目录
   :local:
   :depth: 2

论文信息
--------

摘要
----

关键词
------

符号与缩写
----------

1 引言
------

...

参考文献
--------

完整引用
--------

收录信息见 :ref:`WOEAI 学术成果页对应条目 <<publication_ref>>`。
```

页面末尾必须保留“完整引用”小节，并链接到：

```rst
:ref:`WOEAI 学术成果页对应条目 <<publication_ref>>`
```

可在末尾加入“相关论文精解”站内导航，但只能链接已经存在的 `docs/source/paper-notes/*.rst`。

## 覆盖自查

交付前必须做一次覆盖自查。不要把自查表放进公开 RST 页面；可以在回复或私有工作记录中报告。

自查表必须包含：

- PDF 总页数与正文页范围。
- 处理过的章节列表，逐项写出“已翻译 / 不适用 / 缺源文件”。
- 图号覆盖：例如 `Fig. 1-12 all included`，或列出缺失图号与原因。
- 表号覆盖：例如 `Table 1-5 all included`，或列出缺失表号与原因。
- 公式编号覆盖：例如 `Eq. (1)-(39), (A1)-(D6) all included`，或列出缺失公式与原因。
- 参考文献覆盖：说明参考文献列表是否完整保留。
- 附录覆盖：说明每个附录是否完整翻译。
- 全文搜索确认公开 RST 中不存在 `pending`、`TODO`、`略`、`此处省略`、私有绝对路径、`\tag{`。

如发现漏译，必须先补齐再交付。

## 完成后运行

至少运行：

```bash
python3 tools/publications/artifacts.py --check
python3 scripts/check-public-safe-content.py
./scripts/check-docs.sh
```

如果只修改了提示词本身，也至少运行：

```bash
python3 scripts/check-public-safe-content.py
git diff --check
```
